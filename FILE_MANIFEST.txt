FILE 2: StudentDatabase.h
────────────────────────────────────────────────────────────────────────────────
Type: C++ Header File
Purpose: Database classes for managing student records
Language: C++17
Key Classes:

A. IIITStudentDatabase
   Purpose: Manages IIIT-Delhi students with string-based course codes
   Template: Student<std::string, std::string>

   Key Methods:
   • addStudent(student) - Add new student (thread-safe)
   • updateStudent(rollNo, name, branch, year) - Update student info (Admin)
   • findStudent(rollNo) - Find by roll number (thread-safe)
   • readFromCSV(filename) - Load 3000 records from CSV file
   • parallelSort() - Sort using 2 threads with timing logs
   • buildCourseGradeIndex() - Create indexed lookup structure
   • getStudentsByGradeAndCourse(course, minGrade) - Fast O(1) lookup
   • getStudentsInsertionOrder() - Get students in insertion order (iterator)
   • getSortedStudents() - Get sorted students (iterator)
   • displayAll() - Display all students

B. IITStudentDatabase
   Purpose: Manages IIT-Delhi students with integer-based course codes
   Template: Student<std::string, int>
   Methods: Same as IIITStudentDatabase but for int course codes

Assignment Requirements:
  Question 2 (Multi-University System): ✓ Both IIIT and IIT databases
  Question 3 (Parallel Sorting): ✓ 2 threads, no race conditions, timing logs
  Question 4 (Iterators): ✓ Insertion order and sorted order iterators
  Question 5 (Fast Lookup): ✓ Course-grade index with O(1) access

Thread Safety:
  • All operations use std::mutex (dbMutex)
  • All accesses use std::lock_guard for RAII
  • No race conditions possible
  • Concurrent read-safe
  • Exclusive write access

Parallel Sorting Algorithm:
  1. Copy data to sortedStudents
  2. Create 2 threads
  3. Thread 1: Sort first half [0, mid)
  4. Thread 2: Sort second half [mid, end)
  5. Join both threads
  6. Merge two sorted halves
  7. Log time for each thread and total

FILE 4: main.cpp
────────────────────────────────────────────────────────────────────────────────
Type: C++ Source File
Purpose: Application entry point and user interface
Language: C++17
Lines of Code: ~450

Global Objects:
  • IIITStudentDatabase iiitDB - Main database for IIIT students
  • IITStudentDatabase iitDB - Database for IIT students (ready for expansion)
  • AuthManager authManager - Manages user authentication

Menu Structure:

  Main Menu:
    1. Login as Admin
    2. Login as Student
    3. Exit

  Admin Menu (7 Options):
    1. Load student data from CSV
       - Loads 3000 records from students_data.csv
       - Supports multiple current and previous courses
       - Parses grades from course:grade format

    2. Add new student
       - Input: Roll number, Name, Branch, Starting Year
       - Creates new Student object
       - Adds to database thread-safely

    3. Update student information
       - Input: Roll number to find, new Name, Branch, Year
       - Modifies existing student record
       - Thread-safe operation

    4. Perform parallel sorting
       - Runs 2 threads for sorting halves
       - Logs Thread 1 execution time
       - Logs Thread 2 execution time
       - Logs total sorting time
       - Merges sorted halves

    5. Find students by grade in course
       - Input: Course name (e.g., "OOPD")
       - Displays all students with grade >= 9 in course
       - Uses O(1) indexed lookup
       - No linear search

    6. Display all students
       - Shows first 10 students (displays full if < 10)
       - Shows: Roll number, Name, Branch, Year, Courses, Grades
       - Format: Clear and readable with separators

    7. Logout
       - Ends admin session
       - Returns to main menu

  Student Menu (4 Options):
    1. View all students (insertion order)
       - Displays first 10 students in insertion order
       - Shows complete student information
       - Read-only access

    2. View sorted students
       - Shows first 10 students in sorted order
       - Sorted by roll number (ascending)
       - Read-only access

    3. Search student by roll number
       - Input: Roll number to search
       - Displays matching student if found
       - Shows complete information

    4. Logout
       - Ends student session
       - Returns to main menu

Supporting Functions:
  • displayMainMenu() - Shows main menu options
  • displayAdminMenu() - Shows admin menu options
  • displayStudentMenu() - Shows student menu options
  • adminLogin() - Authenticates admin user

Type: CSV Data File

  • RollNo: Student roll number (MT25000 to MT27999)

  • Branch: CSE, ECE, ME, CE, or EE
  • StartingYear: 1 to 4
  • CurrentCourses: Comma-separated course names
  • PreviousCourses: Course:Grade pairs separated by commas

Rows: 3001 (1 header + 3000 data rows)
Size: ~150-200 KB
Format: RFC 4180 compliant CSV
Parsing: Custom parser in StudentDatabase.h
